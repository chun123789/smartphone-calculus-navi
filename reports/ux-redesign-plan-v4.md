# UXリデザイン実行計画 v4

- 作成日: 2026-03-02
- 根拠: `reports/competitive-design-audit-2026-03-02.md`

## 目的

- 「ダサい」印象を解消し、競合水準の導線明瞭性と操作体験を両立する。
- 高校生がスマホで20秒以内に「次に読む記事」を決定できるUIへ更新する。

## 実装計画

1. IA再設計
- トップを4ブロック固定（30秒で選ぶ / 最初の1本 / 今日の10分 / 最近の更新）
- 単元導線は補助ブロックで常時表示

2. 情報密度調整
- 記事導線は「次の1本」を先出し
- 関連6リンクは折りたたみ表示

3. 可視化体験の統一
- 再生/停止、リセット、意味色凡例を全インタラクティブで共通化
- テーマに追従する可視化カラートークン導入

4. ビジュアル刷新
- ヘッダーを学習アプリ風（ロゴバッジ+タブバー）へ変更
- カード/ステップ/ピルUIで構造を明確化
- 視認性を最優先したライト/ダーク両テーマ

5. 品質保証
- 既存数値テストを維持
- 出力HTMLの順序/A11y/重複導線を自動検証
- build/test/lintを必須ゲートにする

## 完了条件

- `npm test`, `npm run build`, `npm run lint` が通る
- トップ導線の重複カードを排除
- 主要記事2本で導線順序が固定される
- ローカルプレビューでモバイル想定表示を確認できる
